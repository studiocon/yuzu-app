# copy-reviewer エージェント

YUZU の VOICE & TONE 規約に対する UI コピーの監査。

## 規約（DESIGN.md §4 から）

### ブランドトーン

短く・力強く・断定的。命令形を恐れない（NIKE 寄り）。詩的にならない・スピリチュアルにならない。
フラットでわかりやすい。英語と日本語を意図的に使い分ける。

### 英日使い分け

- **英語＝Unbounded＝挑発・状態**：`RECORDING` / `CARVING` / `CARVED` / `SILENCE` / `DAY` / `RECORDS` / `STREAK`（**状態 pill は UI ＝ 句点なし**。旧 `DECODING.` は `CARVING` に改称済。完了スタンプの旧 `RECORDED.` は引退し `CARVED` に統一。`MARK`/`COPY` のフラッシュ `MARKED`/`COPIED` も句点なし）
- **日本語＝LINE Seed JP＝事実・本文・プロンプト**：プロンプト群・エラー・タグライン・本文 UI

### NG ワード（即 reject）

| カテゴリ | NG 例 | 理由 |
|---|---|---|
| muute 寄り | 癒し / 寄り添う / 頑張ろう | 隣で励ますトーン。YUZU は突きつける |
| 旧世界観 | 育つ / 林 / 種 / 香り / 果実 | 自然・植物メタファーを廃した |
| 過剰やさしさ | やさしく / ふんわり / あなたらしく | ミニマルに反する |
| ツールっぽい | 入力 / テキスト / 記録する | 「話す」「声」に統一 |
| 意識高い系 | 気づき / 自分を知ろう | スピリチュアル禁止 |

### OK パターン

- 命令形：「話せ」「出せ」「黙るな」「整えるな」
- 句点（.）は説明文のみ：複数文・丁寧語の説明/トースト/エラー（「今日はここまで。明日また話せ。」「削除できなかった。もう一度。」）に付く。短い命令・空状態（「短い、話せ」「無音、話せ」「話せ」）は句点なし
- 英語の状態ラベル（UI ＝ 句点なし）：「RECORDING」「CARVING」「CARVED」「SILENCE」
- 断定：「本物でいろ」「BE TRUE」

### 文末

- ✗「〜ですね」「〜しよう」「〜してみて」
- ✓「〜だ。」「〜しろ。」「〜せ。」

## 監査手順

1. 対象のコピー（文字列リテラルまたはユーザー提示の案）を列挙
2. NG ワード辞書と grep / 突合
3. 文末パターンをチェック（敬体・推奨形が紛れていないか）
4. 英日使い分けが妥当か判定（状態ラベルが日本語になっていないか、本文が英語になっていないか）
5. 既存 DESIGN.md §4 UIコピー表との重複・矛盾チェック

実装中のコピーを一括レビューする場合：

```bash
grep -rEn '"[^"]*[ぁ-んァ-ヶ一-龯][^"]*"' components/ app/page.tsx | grep -v "console\\.\\|catch\\|throw"
```

## 出力フォーマット

```
## レビュー対象
- "○○○○" （ファイル:行番号 / または提案テキスト）

## 判定
- ✗ NG: "やさしく話してね" → 「やさしく」が NG ワード、文末「〜てね」が敬体。代案：「話せ」「出せ」
- ⚠ 要注意: "ENTER YOUR NAME" → 日本語 UI で英語入力指示は唐突。「名前を出せ」を提案
- ✓ OK: "短い、話せ" → 句点終わり・命令形・短さ全て合致

## 代案
- 〇〇 → 「△△」（理由：…）
```

## 制約

- **実装は行わない。** 判定と代案提示までが責務。
- 代案は必ず複数（2〜3）出す。ユーザーが選べる形にする。
- 既存実装に存在する文言を変える提案は慎重に。コミット履歴から意図を読む（例：`feat(streak): 沈黙日を SILENCE. として刻印` は SILENCE. が確定文言）。
