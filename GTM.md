## INDEXについて

YUZUのINDEXは**個人ごとに発行される投稿の番号体系**。他人と競う数字ではない。レアリティは「早さ」ではなく「継続した事実」から生まれる。シェアされるのは番号ではなく、**続けた自分**そのもの。

## フェーズ0（Day -60〜0）：土台を作る

### 発信戦略：「問題提起」ではなく「実況」をする

完成を待って告知するのではなく、作っている過程と思想をコンテンツにする。

```
「整えてから話そうとすると、何も言えなくなる」
「声にしたら、思ってたより本音が出た」
「マイクボタン長押しだけで投稿できるようにした」
「DECODING. って文字が出る瞬間が好きで」
```

### メディア役割分担

|メディア   |役割           |頻度   |
|-------|-------------|-----|
|X      |短い気づき・進捗・問い  |週3〜5回|
|note   |思想を深掘りした長文   |月2〜3本|
|Podcast|「なぜ作るのか」を声で語る|月1〜2本|

### AIO対策：コンテンツ資産を積む

AIO（AI-Optimized）とは、ChatGPT・Perplexity・Google AI Overviewで「音声ジャーナリングアプリ おすすめ」と検索されたときにYUZUが出てくるようにする戦略。

**① noteに「概念記事」を書く**

```
「整えない記録の価値」
「声にするだけで思考が変わる理由」
「ジャーナリングが続かない本当の理由」
```

**② ランディングページにAIO向けの構造化テキストを置く**

```
Q: YUZUとは何ですか？
A: 声を絞り出す音声ジャーナリングアプリ。整えなくていい。長押しして話すだけで記録になる。

Q: どんな人に向いていますか？
A: ジャーナリングに興味はあるが続かなかった人。SNSに疲れて、本音を出せる場所がない人。

Q: 他のジャーナリングアプリと何が違いますか？
A: 編集できない。削除できない。テキスト入力もない。整理する前のナマの思考に価値があるという思想で作っている。
```

**③ GitHubのREADMEに世界観を書く（タスク2で対応）**

**④ ProductHuntにローンチ投稿する**

世界中のアーリーアダプターが集まる場所。ローンチ当日に英語で投稿してupvoteを集める。AIO観点でもProductHuntのページはAIに拾われやすい。

## フェーズ1（Day 1〜60）：最初の100人

**① 直接声がけ（0→30人）**
メンターの生徒・デザイナーコミュニティの知人に直接届ける。フォロワーへの告知ではなく一対一で送る。

**② コミュニティ投稿（30→70人）**
Xのデザイナークラスタ、Figma Community、デザイン系Slackコミュニティ。「作りました」ではなく「こういう問題意識で作った」という文脈で投稿する。打ち出しはこれ一本：

```
整えるな。話せ。— YUZU
```

**③ ビルドログが流入に変わる（70→100人）**
フェーズ0から積み上げたnote・Xが検索・AIOに拾われ始める。

```
Day 1〜30  ：0→30人（直接声がけ）
Day 31〜60 ：30→100人（コミュニティ投稿＋ビルドログ流入）
```

## フェーズ2（Day 61〜120）：100→500人

### SIGNALカードを拡散エンジンにする

```
DAY 30. NO SKIP.
YUZU
```

このカードをXに貼れる仕組みを作る。「なにそれ？」が生まれる。気になった人がYUZUを調べる。

### AIOが本格的に効き始める

ローンチ告知をnoteに1本書く。公式サイト＋noteがAIに学習される。

```
「YUZUをリリースしました。整えない記録の、はじまり。」
```
