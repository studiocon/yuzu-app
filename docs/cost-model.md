# YUZU 原価モデル（1ユーザーあたり変動費）

> このドキュメントは **技術的な変動費（STT / LLM）の単価と、1ユーザーあたり月間原価** を扱う。
> **価格・プラン設計・GTM・損益分岐は含めない**（[PRD.md](../PRD.md) / CLAUDE.md の方針どおり Notion に集約）。
> ここに書くのは「どの機能をサーブするのにいくらかかるか」＝**有料化ラインを引くときの技術的根拠**。
>
> 初版: 2026-07-11。データ元は Anthropic コンソール CSV（2026-06 / 2026-07）と ElevenLabs クレジット CSV（2026-04〜2026-07）。

---

## 1. スコープと前提

- **変動費のみ**を対象（Supabase / Vercel の固定費・無料枠は別管理）。
- 原価ドライバは 3 系統：
  1. **STT** — ElevenLabs Scribe v2（録音 → 文字起こし）
  2. **感情スコア** — Anthropic（[lib/sentimentScore.ts](../lib/sentimentScore.ts)、投稿ごと）
  3. **インサイト生成** — Anthropic（[lib/reports.ts](../lib/reports.ts) の週次/月次レポート、[app/api/insights/themes/route.ts](../app/api/insights/themes/route.ts) の PATTERN/テーマ）
- **ハードキャップが効いている**：[lib/constants.ts](../lib/constants.ts) の `MAX_DAILY_SESSIONS = 1`（1日1録音）/ `MAX_RECORD_MS = 1分`。これがコストの上限を規定する。**この2値を緩めると STT が線形に増える**。
- **STT と Claude はコスト構造が違う**：
  - **STT（ElevenLabs）＝ 無料枠10,000クレジット/月の固定プール**（全ユーザーが1つの `ELEVENLABS_API_KEY` を共有）。枠を超えるまで実質¥0、超えたら定額プランに段差で乗る **ステップ関数**。
  - **Claude（Anthropic）＝ 従量課金で1ドル目から線形**。売上ゼロでも比例して増える。
  - → **スケール時に効いてくるのは Claude 側**。STT は当分安い（§6参照）。

## 4.5 ElevenLabs 無料枠のキャパシティと「課金が必要になるタイミング」

**STT は 10,000クレジット/月の固定プールを全ユーザーで共有**（1本の API キー）。1分の録音 ≈ **22クレジット**なので：

- **無料枠 10,000クレジット = 約450分/月 = 約450本の1分録音**（全ユーザー合算）

これを実ユーザー数に落とすと（1人が月に何分録音するかで決まる）：

| 実ユーザーの使い方 | 1人あたり/月 | **無料枠で収容できる人数** |
|---|---|---|
| ヘビー（30分/月） | 660cr | **~15人** |
| 標準（15分/月） | 330cr | **~30人** |
| ライト（10分/月） | 220cr | **~45人** |

→ **無料公開しても STT だけなら 15〜45 MAU までは完全無料**。それを超えたら段差で有料プランへ：

| 次の一手 | 月額 | クレジット | 無料枠比 | 収容MAU目安(標準) |
|---|---|---|---|---|
| Starter | $6 | 30,000 | 3× | ~90人 |
| Creator | $11〜22 | 121,000 | 12× | ~360人 |
| Pro | $99 | 600,000 | 60× | ~1,800人 |
| ElevenAPI 従量 | — | $0.22/時 | — | 上限なし・線形 |

**現状（あなた1人・開発コミで月2,700〜3,100cr = 27〜31%）→ 当面は無料 Free で問題なし。**
**課金の初トリガーは「STT合算が月10,000crを超える瞬間」= ざっくり 15〜45 MAU 到達時**。ただし乗り換え先が月$6〜22と安いので、STTは事業を圧迫しない。**先に効いてくるのは Claude（1ドル目から線形）**。

## 5. 隠れコスト（放置すると単価が上振れる要因）

1. **PATTERN/テーマの入力が全投稿履歴に比例** — [app/api/insights/themes/route.ts](../app/api/insights/themes/route.ts) は履歴全体を入力に使うため、長期ユーザーほど1再生成のトークンが肥大する。→ **直近N投稿への入力窓制限**が必要。
2. **上限緩和は即STT増** — `MAX_DAILY_SESSIONS` / `MAX_RECORD_MS`（[lib/constants.ts](../lib/constants.ts)）を緩めると STT が線形に増える。緩和は課金プラン側でのみ行う。
3. **レポートの再生成** — 失敗時リトライやバックフィルで多重生成が起きるとレポート原価が膨らむ（`report_jobs` の冪等性で抑止中）。

## 6. 原価の「高い所」＝有料化ラインの技術的根拠

**課金の壁を置くべき場所＝原価が発生する場所**。原価は下流の Claude インサイトに集中しており、STT（録音＝コア体験）は無料維持しても安い。

| 機能 | 原価 | 無料 / 有料の目安 |
|---|---|---|
| LOG一覧・STREAK | ~0 | **無料**（DBのみ） |
| 投稿ごとの感情pill | 激安（64tok） | **無料** |
| 録音 + STT | ¥40前後/月（1分/日） | **無料維持が妥当**（コア体験・安い） |
| 週次・月次レポート | 最大原価（reports.ts） | **有料へ** |
| PATTERN/テーマ | 履歴で肥大 | **有料へ** |
| ヒートマップ/words 高度版・長期センチメント時系列 | 中 | **有料へ** |

→ **Claude 系インサイトを有料へ寄せると、無料ユーザーの変動費が「STT + 感情pill」＝ ¥40前後まで落ちる**。これが無料公開の持続可能性の生命線。

> プラン構成・価格（¥）・損益分岐・課金率の設計は **Notion に集約**（このリポジトリには置かない）。

## 7. 原価削減レバー（価格設計と独立に実施可能）

1. **感情スコアを Haiku 4.5 に**（$1/$5、Sonnet比 3倍安）— 64tokの単純判定なので品質影響は小さい。[lib/sentimentScore.ts](../lib/sentimentScore.ts) のモデルIDを差し替え。
2. **PATTERN入力の窓制限** — 全履歴 → 直近N投稿。長期ユーザーのトークン肥大を止める。
3. **レポートの一部 Haiku 化 / effort 調整** — 週次など軽い期間は下位モデルで十分か検証。
4. **STT ティアの見直し** — ElevenLabs の実請求で $/分を確定し、ボリューム次第で上位ティアの単価に載せ替え。
