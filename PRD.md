> BE TRUE / 本物でいろ

YUZUは、声を絞り出すサービス。整えなくていい。うまく話さなくていい。長押し。話せ。あなたの声はそのまま記録になる。

## 1. ブランド

- **名前**：YUZU
- **タグライン**：BE TRUE
- **タグライン（日）**：本物でいろ
- **コアバリュー**：Raw. Real. You.
- **ミッション**：まだ知らない自分に、出会わせる。

### エレベーターピッチ

YUZU という名前は、果物の柚子から来ている。柚子は絞るか、削るか、熱を加えなければ香りが出ない。出さなければ、ただの黄色い球体だ。声も同じだ。外に出さなければ、自分は自分のまま留まる。

「BE TRUE」は、シェイクスピアの「To thine own self be true.」に由来する。YUZU が届けるのは命令ではなく、許可だ。あなたはすでに本物だ。ただ、出していないだけ。

YUZUは、声を絞り出すサービス。
整えなくていい。うまく話さなくていい。
長押し。話せ。
あなたの声はそのまま記録になる。

## 2. コンセプト・世界観

### コアバリュー

```
Raw  → 加工しない・整理しない・ナマのまま
Real → 本物の自分・嘘のない声
You  → 主役はあなた自身
```

### ポジショニング

```
muute → 心を癒す場所（やさしい・内省的・自己理解）
YUZU  → 本物を絞り出す装置（クール・挑発的・記録的）
SNS   → 他人に見せる場所
YUZU  → 自分を晒す場所
```

### 世界観「THE RECORD」

YUZU に庭はない。林もない。育つ比喩もない。
あるのは **記録（RECORD）** だけ。

```
声を出す → 信号（SIGNAL）になる
記録する → MARK（刻印）として残る
積み重なる → INDEX（番号体系）として並ぶ
```

声は加工されない信号。記録されることでしか自分は見えない。
装飾を削ぎ落とすこと自体が "Raw" の表現になる。

### MARK / INDEX

**抽象記号としての MARK** を導入する。

```
1投稿 = 1 MARK
表現   = 番号（#0001, #0002 …）＋ 日時
キャラ絵なし。マスコットなし。
```

`MARK` が積み重なった全体を **`YUZU INDEX`** と呼ぶ。
これは将来的にグラフィック展開・グッズ・アパレル等で
番号体系のビジュアル資産として再利用できる。

- 「続けてる自分」がかっこいいと思いたい
- 習慣になるきっかけが欲しい

## 4. 機能要件（MVP v1）

### 基本構成

- **3タブ構成**：タイムライン / はなす（デフォルト）/ マイページ。下部に iOS 26 Liquid Glass 風のフローティングタブバー。
- **ゲストモード**：投稿・閲覧・リアクション可能（cookie で sessionId を保持）。

### 投稿

- マイクボタン長押し中のみ録音（MediaRecorder）
- 離した瞬間に自動で STT（ElevenLabs Scribe v2）送信
- テキスト変換後、自動投稿（生音声は保存しない）
- 編集・削除不可
- テキスト入力欄なし
- NGタグ（[background noise] 等）を除去してから保存
- 5文字未満の発話は破棄しフィードバック表示

### マイページ（デフォルト画面）

- ニックネーム登録・編集（未登録はランダム果物名）
- 果物絵文字アイコン（既存資産・MVP では維持。新世界観のIPは MARK / INDEX で別表現）
- スタッツ表示（英語ラベル）：
  - **DAY ○** — 登録からの日数
  - **○ RECORDS** — 投稿数
  - **○ DAYS. NO SKIP.** — 連続投稿日数
  - **○ CHARS** — 総文字数
  - **○ MIN** — はなした時間（投稿数 × 平均30秒）
- 感情分析チャート（Claude API・折れ線グラフ）
- 投稿一覧（新着順・各カードに `#0042` のような連番表示）

### 投稿促進プロンプト

マイクボタン上にランダム表示（毎セッションで変わる）：

```
何が本当だ？
言ってないことは？
今日、誰に嘘をついた？
本当はどう思った？
逃げてることは？
怒ってるのは何にだ？
誰にも言えないことを。
整えるな。話せ。
1分でいい。出せ。
黙ってる場合か？
```

### マイクボタン

- 右下フローティング（FAB）
- 全画面から常時表示
- タップでモーダル起動

### モーダル（録音画面）

- 背景：**オフホワイト単色** `#FAFAF5`
- 上半分：アニメーションエリア
  - 待機中：**浮遊するドット**（既存 `float-dot` 維持）
  - 録音中：**音量に反応する波形**（既存波形アニメ維持）＋ 同心円リング
  - 変換中：ドットが中心に集まり消える
- 下部：マイクボタン（正円。歪んだ楕円シェイプは廃止）
- 録音完了後：**`RECORDED.`** ＋ 投稿テキスト ＋ 連番 `#0042` がスタンプされる → ホームに戻る

### UIコピー

英語＝状態ラベル（Unbounded）／日本語＝プロンプト・本文・エラー（LINE Seed JP）として使い分ける。

| 場所 | コピー |
|---|---|
| 待機（マイクボタン下） | 長押し。話せ。 |
| 録音中 | RECORDING. |
| 変換中 | DECODING. |
| 投稿完了 | RECORDED. |
| タイムライン空 | 何も無い。話せ。 |
| ニックネーム未設定 | 名前をつけよう |
| 投稿数 | ○ RECORDS |
| 連続日数 | ○ DAYS. NO SKIP. |
| 登録日数 | DAY ○ |
| 音声なし | 声が聞こえなかった。もう一度。 |
| 短すぎ | 短い。もう一度。 |

## 11. Eval（品質基準・テストケース）

> PRD の要件変更時は、対応する Eval ケースも同時に更新すること。

### STT（ElevenLabs Scribe v2）

**成功基準**：3秒以内・日本語として読める・NGタグ除去済み

- 30秒音声 → 句読点ありテキスト・3秒以内
- 無音・環境音 → 保存されない・フィードバック表示
- NGタグ → 除去されてから保存される
- 5文字未満 → 保存されない・フィードバック表示

### 感情分析（Claude API）

**成功基準**：自然な日本語・200文字以内・断定的ネガティブ表現なし

- ポジティブ投稿10件 → 前向きな要約
- ネガティブ投稿10件 → 断定的なネガティブ表現にならない
- NGケース：「あなたはネガティブです」→ 不合格

### 投稿保存（Upstash Redis）

**成功基準**：投稿後1秒以内にタイムラインに反映

- 投稿 → タイムラインの先頭に表示・連番 `#NNNN` が付与
- 連続投稿（5秒以内2回）→ 両方保存

### モーダルUX

**成功基準**：録音→投稿→ホーム復帰が自然なフロー

- 待機中：浮遊ドットが軽快に動く
- 録音中：波形アニメーションが音量に反応する
- 完了後：`RECORDED.` ＋ 投稿テキスト ＋ 連番がスタンプされる
- ホームに戻る：マイページ／タイムラインに反映されている

### コピートーン

**成功基準**：NGワード（癒し・寄り添う・育つ・香り・林・種・やさしく・ふんわり）がUI上に登場しないこと
