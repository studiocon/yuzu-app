# YUZU — PRD

> BE TRUE / 本物でいろ

YUZUは、声を絞り出すサービス。整えなくていい。うまく話さなくていい。長押し。話せ。あなたの声はそのまま記録になる。

## 1. ブランド

- **名前**：YUZU
- **タグライン**：BE TRUE
- **タグライン（日）**：本物でいろ
- **コアバリュー**：Raw. Real. You.
- **ミッション**：まだ知らない自分に、出会わせる。
- **ドメイン**：`yuzu.style`
  - **選定理由**：ライフスタイルブランドとして育てる方針に基づき選定。`yuzu.app` は取得済みの他者が保有。`yuzu.style` はライフスタイルブランド宣言そのものをドメインで体現する。
  - **サブドメイン方針**：アプリは `app.yuzu.style`、公式サイトは `yuzu.style`、スタジオこんのサイトは別ドメインで運営。

### プラットフォーム方針

- **Web版（yuzu.style）**：公式LP・ダウンロード導線のみ。アプリ機能はWebでは提供しない。今野が開発確認用として使い続ける予定。
- **iOS**：メインプラットフォーム。App Store配信。
- **Android**：iOS安定後に検討。検証端末の調達が必要。

### エレベーターピッチ

YUZU という名前は、果物の柚子から来ている。柚子は絞るか、削るか、熱を加えなければ香りが出ない。出さなければ、ただの黄色い球体だ。声も同じだ。外に出さなければ、自分は自分のまま留まる。

「BE TRUE」は、シェイクスピアの「To thine own self be true.」に由来する。YUZU が届けるのは命令ではなく、許可だ。あなたはすでに本物だ。ただ、出していないだけ。

YUZUは、声を絞り出すサービス。
整えなくていい。うまく話さなくていい。
長押し。話せ。
あなたの声はそのまま記録になる。

## 2. コンセプト・世界観

### 2.1 コアバリュー

```
Raw  → 加工しない・整理しない・ナマのまま
Real → 本物の自分・嘘のない声
You  → 主役はあなた自身
```

### 2.2 ポジショニング

```
muute → 心を癒す場所（やさしい・内省的・自己理解）
YUZU  → 本物を絞り出す装置（クール・挑発的・記録的）
SNS   → 他人に見せる場所
YUZU  → 自分を晒す場所
```

#### 3つの市場の隙間を狙う

1. **ジャーナリング市場（muute / Awarefy）**
   - 共通点：内省・自己理解
   - 違い：YUZUは「癒し」を持たない。挑発・命令・信号。muuteが "肩を抱く" ならYUZUは "胸ぐらを掴む"。

2. **SNS市場（X / BeReal）**
   - 共通点：思考や日常を外に出す
   - 違い：YUZUは他人に見せるためのものではない。晒しても評価されない構造。

3. **音声記録市場（Voice Memo / Otter / Untold）**
   - 共通点：音声で記録する
   - 違い：YUZUは "態度" を持つ。記録ツールではなく、自己と対峙する装置。

### 2.3 世界観「THE RECORD」

YUZU に庭はない。林もない。育つ比喩もない。
あるのは **記録（RECORD）** だけ。

```
声を出す → 信号（SIGNAL）になる
記録する → MARK（刻印）として残る
積み重なる → INDEX（番号体系）として並ぶ
```

声は加工されない信号。記録されることでしか自分は見えない。
装飾を削ぎ落とすこと自体が "Raw" の表現になる。

## 3. ターゲットユーザー

### 3.1 ターゲット階層

#### プライマリ：本音発信渇望層（Voice-Hungry）

- **定義**：20代〜30代。SNSに本音を出したいが、人目を気にして出せない層。
- **特徴**：
  - X鍵垢・Instagram裏垢を持っている
  - SNS疲れを感じている（Z世代の51%が該当 / SHIBUYA109調査）
  - 自分の発信が共感を得られるか不安がある（NRI調査）
  - muute・X・noteを使い分けているが、どれにも本音を出しきれていない
- **YUZUに来る動機**：誰にも見られず、誰にも評価されない場所で、整える前の声を残したい。

#### セカンダリ：自己理解探求層（Self-Discovery）

- **定義**：内省したいが、文章を書くハードルが高い層。
- **特徴**：
  - ジャーナリングを試したが3週間で挫折した経験がある
  - 「自分が何を考えているか分からない」自覚がある
  - 瞑想アプリ・モーニングページ・5年日記などを試している
  - muute・Awarefy・Day Oneを試したが続かなかった
- **YUZUに来る動機**：書く構えを取らずに、声で思考を外に出したい。

#### ターシャリ：クールカルチャー層（Street / Culture）

- **定義**：カルチャー感度の高い20代〜30代。NIKE・Patta・A-COLD-WALL\*・Supreme・Off-Whiteなどに親和性がある層。
- **特徴**：
  - 「やってる感」を嫌う一方で、本物の習慣には価値を置く
  - メンタル系・ウェルネス系アプリは「ダサい」と感じる
  - StravaやNIKE Run Clubは続けている
  - YUZU INDEX・YUZU Wearに反応する
- **YUZUに来る動機**：「これ使ってる」と言えるブランドだから。INDEX番号やSTREAKで自分のストイックさを可視化したい。

### 3.2 除外ターゲット

YUZUが意図的に取りに行かない層。

- **30代後半以上のウェルネス層** → muuteで満たされている
- **本格的メンタル不調層** → 専門治療（Awarefy・カウンセリング）が必要
- **きれいな日記を書きたい層** → 編集削除不可と相性最悪
- **承認欲求が主動機の層** → いいね・フォローがないと続かない

### 3.3 ペルソナ

#### ペルソナA：ナナミ（27歳・編集者）「言葉を持て余す内省者」

- **属性**：都内・出版社勤務・独身・千駄ヶ谷在住
- **行動特性**：X鍵垢、Instagram裏垢、TikTok閲覧専。ずとまよ、ベイクルーズ、ATELIER NAVE
- **悩み**：思考を文章にする職業なのに、自分の本音は書けない。"言葉になる前の感情" を逃している感覚がある。
- **インサイト**：「きれいに書こうとした瞬間、嘘になる」のを知っている。だから書けない。
- **利用シーン**：仕事帰り、駅から家までの15分。整える前の自分を、捨てずに残しておきたい。

#### ペルソナB：リク（24歳・グラフィックデザイナー）「クールに自分を晒したい挑発者」

- **属性**：フリーランス・代々木上原・MacBook Air
- **行動特性**：Patta, NIKE SB, A-COLD-WALL\*, Travis Scott。『ブルータス』『POPEYE』『IDEA』『装苑』を読む
- **悩み**：ジャーナリングは陰キャっぽい。でも自分と向き合う時間は欲しい。「やってる感がダサい」「本物かどうか」が判断軸。
- **インサイト**：「これ使ってる」と言えるブランドなら、する。
- **利用シーン**：深夜の制作中、作業の合間。アイデアとも愚痴ともつかない声を吐き出して、INDEX番号を増やすのが快感。

#### ペルソナC：ユイ（31歳・人事）「ジャーナリングに挫折した自己探求者」

- **属性**：中堅メーカー人事・横浜・結婚2年目
- **行動特性**：muute、5年日記、モーニングページ、瞑想アプリ — すべて3週間で挫折
- **悩み**：「ちゃんと書こう」としてしまう。
- **インサイト**：続かないのは意志が弱いからではない。「ちゃんとやろう」という構えが続かない原因だと薄々気づいている。
- **利用シーン**：朝のメイク中、洗濯機を回している間、寝る前のベッド。両手が塞がっている時間に、声だけで残せることが革命的。

### 3.4 インサイト

#### インサイト1：整理した瞬間、本物じゃなくなる

- **主張**：文章は推敲する。推敲とは無意識のうちに "見せたい自分" に近づける作業。
- **根拠 / 背景**：声は推敲できない。口から出た瞬間、嘘がつけない。これがYUZUの最大の発明。

#### インサイト2：キーボードは構えの装置

- **主張**：スマホを開く → アプリ起動 → キーボードが立ち上がる、この時点で人は "書く構え" に入っている。
- **根拠 / 背景**：長押し録音は、構えを与えない。思考が言語化される前の、皮膚感覚の声が拾える。

#### インサイト3：日記が続かないのは、形式があるから

- **主張**：形式（プロンプト、テンプレ、AIの励まし）があると、人は "正解" を探し始める。
- **根拠 / 背景**：YUZUは形式を持たない。プロンプトすら「整えるな。話せ。」と命令形で剥がす。既存のジャーナリングアプリ（muute等）は、ポジティブ報告を促すフィードバックでユーザーを追い詰めるケースがある。YUZUはフィードバックで誘導しない。

#### インサイト4：やってる自分がイケてる、と思いたい

- **主張**：NIKE TRAINING CLUB、Strava、NIKE Run Clubが流行ったのは「やっている」状態が可視化できたから。
- **根拠 / 背景**：YUZUのSTREAK・DAY・RECORDSはストイックさの可視化装置。muuteの "やわらかい数字" とは正反対の方向に振る。

#### インサイト5：声は最も親密で、最も嘘がつけない媒体

- **主張**：電話で泣くのは止められない。声には体温がある。
- **根拠 / 背景**：自分の声を再生して聴くのは、文字を読み返すのとは別次元の自己対峙。これがV2以降のAIインサイト読み上げ（TTS）の本質的価値。

### 3.5 ペイン

| ペイン | 既存サービスの状態 | YUZUの解 |
|---|---|---|
| 書く前にやめてしまう | キーボード起動の摩擦 | 長押しだけ |
| 上手く書こうとしてしまう | 編集機能、保存ボタン | 編集削除不可 |
| 続かない | カレンダー型UI、リマインダー | STREAK・INDEX・モノ展開 |
| SNSで本音が出せない | 人目、フォロワー、いいね | 晒しても評価されない構造 |
| 自分の感情が分からない | テキスト分析のみ | 声の波形・トーン（V2以降の余地） |

特に "上手く書こうとしてしまう" は他では解けない。テキスト入力できる時点で、人は構えてしまう。YUZUがテキスト入力欄を持たないのは、思想として正しい。

### 3.6 Jobs To Be Done

#### Functional Job（機能的）：整理する前の思考を、捨てずに残したい

- 移動中・歩行中・両手が塞がっている時に記録したい
- 言語化する手間なく、感情を外に出したい
- 後で振り返れる形でストックしたい

#### Emotional Job（感情的）：整える前の自分を、許可したい

- 本音を出していい場所が欲しい
- "ちゃんとしてない自分" を否定されたくない
- SNSの "見せ場" から退避したい
- 自分と対話している感覚が欲しい

#### Social Job（社会的）：自分を律してる人間でありたい / そう見られたい

- 続けてる自分がかっこいいと思いたい
- "メンタル系アプリ" ではなく "ストイックなツール" として持ちたい
- INDEX番号やYUZU Wearで、努力の可視化・自己表現したい

この三層のうち、最も差別化できるのはSocial Job。muuteは "癒し" 止まりでSocialまで行けていない。YUZUのINDEX→Goods→Wear展開は、Social Jobを満たす設計として機能する。NIKEがランニングを "かっこいいこと" にしたのと同じ構造。

## 4. ブランディング差別化軸

### 4.1 Voice & Tone（muuteとの対比）

| 軸 | muute | YUZU |
|---|---|---|
| 動詞 | 寄り添う・気づく | 話せ・出せ |
| 主語 | あなた | YOU（記号化） |
| 文末 | 〜ですね、〜しよう | 〜だ。〜しろ。 |
| 装飾 | やわらかい絵文字 | 句点（.）終わり |
| 比喩 | 自然・植物・育つ | 信号・記録・刻印 |

### 4.2 ビジュアル

```
muute   : パステル・水彩・余白・キャラクター
Awarefy : ブルー・医療的・グラフ・専門性
YUZU    : 単色オフホワイト + 信号黄 + 黒・グリッド・タイポ駆動
```

参照軸：Swiss International Typographic Style、NIKE、Patta、A-COLD-WALL\*、Off-White、Acne Studios。
これはジャーナリングアプリのリファレンスではなく、ストリート / ハイファッション / モダニズムの参照群。

### 4.3 プロダクトの態度

- **muute** = サポーター（隣で励ます）
- **YUZU** = ミラー（突きつける）

ミラーは答えを返さない。自分の声を、自分が聴く。これだけ。介在しない潔さがブランド。

### 4.4 拡張性

```
muute   → アプリで完結
Awarefy → 法人 / 教育機関へ
YUZU    → IP化・グッズ・アパレル・書籍・音楽
```

YUZU INDEX、YUZU Goods、YUZU Wear、YUZU Story、YUZU Sound という拡張軸を持つ。「投稿日数で番号が解放される」設計は、Supremeのドロップ文化 × Stravaの記録性 × NIKE Runの達成感を統合する構造。アプリ単体ではなく、カルチャーとして展開する。

## 5. 哲学

YUZUは "自己肯定" を一切しない。「あなたは本物だ」と励まさない。ただ「許可する」だけ。評価しないことが、最大の許可。

世のセルフケア系サービスの大半は "あなたを肯定する" モデル。YUZUは "あなたを既に本物として扱う" モデル。前提が違う。

シェイクスピア「To thine own self be true.」（自分自身に本物であれ）の現代的解釈として、YUZUは命令ではなく許可を届ける。

## 6. 機能要件（MVP v1）

### 基本構成

- **2タブ + FAB 構成（順序固定）**：左から `LOG`（**デフォルト**・自分が出した記録）/ `INSIGHT`（AI 解釈：SENTIMENT / WORDS / SIGNAL / PATTERN / REPORTS）。LOG と INSIGHT は **情報の出どころが違う**ため分離（バランス目的の追加ではない）。録音は **独立した RECORD FAB**（タブバー右に並ぶ正円ボタン）で起動し、タブの state を持たない。旧「ホーム」「マイページ（ME）」「PROFILE」「TALK タブ（旧 3 タブ中央のマイク）」は廃止。下部のドックは iOS 26 Liquid Glass 風のフローティング pill（タブ）と FAB が横並び。
- **ユーザー identity は通し番号 `#NNN` のみ**。SNS 機能を持たないため、自己を他人に示すアイコン・名前は不要。ニックネーム登録 / 果物絵文字アイコンは v2 で全廃止。
- **認証必須**（MVP v1 から）：Apple Sign In / Google OAuth / Magic Link（メール OTP）の 3 種。パスワード認証は **作らない**。未ログインユーザーには `/` でオンボーディング（録音 → STT → 「記録する」で LoginModal）を見せ、初投稿はログイン後にコミットされる。
- **未ログイン STT 上限：1 日 1 回**（cookie ベース・[app/api/transcribe/route.ts](app/api/transcribe/route.ts) 内 `ANON_DAILY_STT_LIMIT`）。2 回目以降は **429 `login_required`** を返し、クライアントは自動的に `LoginModal` を開く。cookie 改竄は可能だが「お試し体験」と「課金保護」のバランスとしては必要十分。厳密化は将来 IP rate limit（Upstash 等）で対応する余地を残す。
- **タイムラインはプライベート**（自分の投稿のみ）。他ユーザーの投稿は表示しない（RLS で強制）。

### 投稿

- マイクボタン長押し中のみ録音（MediaRecorder）
- 離した瞬間に自動で STT（ElevenLabs Scribe v2）送信
- テキスト変換後、自動投稿
  - **MVP（Free）**：生音声は破棄。テキストのみ保存。
  - **音声保存はV2有料機能**（LIGHT以上）として実装予定。MVP段階では維持しない。
- 編集・削除不可
- テキスト入力欄なし
- NGタグ（[background noise] 等）を除去してから保存
- 5文字未満の発話は破棄しフィードバック表示

### LOG（自分が出した記録 / 独立タブ・デフォルト）

ヘッダー左上の **ページ名 `LOG.`** が見出しを兼ねる（画面内には別途タイトルを置かない）。上から下へ「抽象 → 具体」で縦に積み、**`RECORDS` で完結する**（PROFILE・INSIGHT を含まない）。

1. **STATS（横3列）** — `RECORDS` / `SINCE` / `STREAK`。
2. **RECORDS** — 自分の RECORD 一覧（新着順）。見出し右に `ALL / PINNED` フィルタトグル1つだけ。`PINNED` は MARK 済みのみ抽出。

設定（`/settings`）はヘッダー右上の歯車アイコンに据え置く。`#NNN`（通し番号）は各 RECORD カード上に表示し、identity として機能する。

### INSIGHT（AI 解釈 / 独立タブ）

LOG とは情報の出どころが違うため別タブ。ヘッダー左上は **`INSIGHT.`**。上から「短期 → 長期」で 4 ブロックを縦に積む。

- **SENTIMENT** — Claude API のセンチメントスコアを折れ線グラフで表示（[components/LongSentimentChart.tsx](components/LongSentimentChart.tsx)）
- **WORDS** — 全投稿から頻出語 20 個を抽出してバブルマップ化（[components/WordBubbleMap.tsx](components/WordBubbleMap.tsx)）。形態素解析は TinySegmenter、配置は d3-hierarchy の pack。タップで弾性バウンス（自バブル + 隣接バブルが距離 delay で連動）。「自分がこんな言葉ばっか使ってたのか」という気づきを与える
- **SIGNAL** — 過去 30 日 × 24 時間 = 720 セルの時間帯ヒートマップ（[components/TimeHeatmap.tsx](components/TimeHeatmap.tsx)）。各セルはその時間帯の投稿文字数合計に応じて opacity 0.2〜1.0 で濃淡。ホバー/フォーカスで `MM/DD HH:00 / N CHARS` tooltip。「俺、夜11時に吐き出しがち」のような自分のリズム発見が狙い
- **PATTERN** — Claude が全投稿を読んで「無意識に繰り返し語っているテーマ」を最大 5 つ抽出・**マインドシェア型ランキング** で表示（[components/RecurringThemes.tsx](components/RecurringThemes.tsx)）。各テーマが「あなたの声のうち何%を占めるか」を黄色の横バーで可視化。バブルマップが「生の単語」なら PATTERN は「意味の塊」。WORDS は浅い・即時、PATTERN は深い・じわじわ
  - 投稿 10 件未満は「もっと話せ、パターンが見えてくる」表示
  - サーバ側でプロセス内キャッシュ（user_id をキー、24h TTL、`post_count` が変われば invalidate）。永続化は [Issue #79](https://github.com/kyotar/YUZU/issues/79) で予定
- **REPORTS** — 週次・月次レポートカードを一覧表示（[components/ReadView.tsx](components/ReadView.tsx)）
- "MORE →" で `/reports` 詳細へ遷移

API は [app/api/insights/words/route.ts](app/api/insights/words/route.ts) / [app/api/insights/heatmap/route.ts](app/api/insights/heatmap/route.ts) / [app/api/insights/themes/route.ts](app/api/insights/themes/route.ts)。集計・型は [lib/wordAnalysis.ts](lib/wordAnalysis.ts) / [lib/heatmap.ts](lib/heatmap.ts) / [lib/themes.ts](lib/themes.ts)（SDK 非依存はクライアントとも共有、Claude 呼び出しはサーバ専用）。Mock mode（`?mock=1`）ではクライアント側で同等の集計または hardcoded 例（PATTERN は `MOCK_THEMES`）を使う。fetch + mock の共通パターンは [lib/useInsightData.ts](lib/useInsightData.ts)。

### MARK（刻印）— v2 常時トグル

YUZU は編集・削除を許さない。その代わりユーザーに **唯一許された能動操作** が MARK。「これは本物だった」と未来の自分から過去の声へ刻印する行為。「お気に入り」「いいね」「ブックマーク」とは呼ばない。

**仕様**
- 各 RECORD カード右端の常時トグルボタン（PushPin / PushPinSlash）。v1 の長押し検出は廃止。
- 黄色枠なし。色の差分（線→塗り、`--ink-muted` → `--yuzu-zest`）のみで状態表現。タッチターゲット 44px。
- `records.marked: boolean` で永続化（[supabase/migrations/0004_mark.sql](supabase/migrations/0004_mark.sql)）
- INDEX `RECORDS` に `ALL` / `PINNED` フィルタ（PINNED = `marked === true` のみ）
- 操作直後にカード右下へ `MARKED.`（Unbounded 700・`--yuzu-zest`）を 0.9s フラッシュ
- PINNED 空表示は `MARK されたものは無い。`
- MARK と編集・削除は別概念。UI 上は鉛筆・ゴミ箱アイコンを使わない

### COPY（一時機能 / ⚠️ 将来削除予定）

オーナー（こんちゃん）は現在ジャーナリングを Notion でも運用している。YUZU で十分に振り返れるようになり **本人が Notion 併用を止めたタイミング** でこの機能は削除する **恒久 UI ではない**。

**削除条件（確定）**: オーナー本人が Notion への移行を完了し、YUZU 単独で運用する判断をしたら次の commit で撤去する。第三者ユーザーの利用率は判断材料にしない（オーナー個人のワークフロー基準）。

**実装メモ**
- RECORD カード右端に Copy アイコンを配置（MARK と同等の低主張アイコン）
- クリックで本文 + `#NNN` + 日時をクリップボードへコピー
- 操作直後に `COPIED.` を 0.9s フラッシュ
- コード上に `// TEMPORARY: Notion移行期間限定...` コメント必須（[components/RecordCard.tsx](components/RecordCard.tsx)）

### INDEX番号

ユーザーごとの投稿通し番号。話すたびに1ずつ増加する個人の記録番号。

**ルール**
- 1始まり・ゼロ埋めなし（`#1` `#247` `#10000`）
- 削除・編集不可のため欠番は発生しない
- 上限：999999（6桁）
- 呼称「インデックス番号」はUI上に出さない。`#` + 数字のみ。

**表示箇所**

| 場所 | 形式 | スタイル |
|---|---|---|
| 投稿カード（タイムライン） | `#N · X時間前` | Unbounded 700 / `--ink-muted` / 11px |
| 投稿完了モーダル | `RECORDED.` の下に `#N` | Unbounded 700 / `--ink` / 32px |
| INDEX 通し番号ヘッダー | `#NNN` | Unbounded 700 / `--text-3xl` / ゼロ埋め3桁 |
| 週次レポート | `WEEK #N` | Unbounded 700 / 18px（登録日からの週数） |
| 月次レポート | `MONTH #N` | Unbounded 700 / 18px（登録日からの月数） |

**INDEX 算出**

`records` テーブルに `index` カラムは持たない。`/api/records` GET / POST が以下で都度算出する：

```ts
// app/api/records/route.ts
index = total_count - position   // 最新（position=0）が最大値
```

編集・削除不可なので欠番は発生しない。WEEK・MONTH は登録日（`profiles.created_at`）からの経過で自動算出（保存不要）。

**Eval**
- 投稿後、完了モーダルに `#N` が表示される
- タイムラインの各投稿に `#N · X時間前` が表示される
- INDEX 通し番号ヘッダー（`#NNN`）が更新される
- `INDEX` `No.` 等の文字列がUI上に登場しない
- ゼロ埋め（`#001`）が発生しない

## 7. 機能要件（V2）

- **Supabase Auth（認証）+ DB導入**（Google / Apple ログイン）
  - iOS：Apple ログイン（必須）+ Google ログイン
  - Android：Google ログインのみ
  - Web：Google ログインのみ
- AIインサイト週次レポート（Claude API）
- ElevenLabs TTS（インサイト読み上げ）
- 公開・非公開選択
- **YUZU INDEX グラフィック展開**（投稿日数連動の番号アンロック）
- **過去ログ連動の挑発プロンプト生成**（Claude API）
  - 録音モーダルを開いた瞬間、固定10種ではなく「その人だけに効く挑発」を1つ生成して表示
  - 例：「3日前、逃げてるって言ってたな。今は？」「先週の怒り、まだ残ってるか？」
  - 直近 N 件（3〜5件）の本音ログを Claude API へ送信し、Voice & Tone 制約（短く・断定・命令形・NGワード禁止）を満たす1文を返す
  - 表示までのレイテンシは固定10種を先に出してから差し替え（チラつかない）
  - 失敗・タイムアウト・投稿0件は固定10種にフォールバック
  - **プライバシー検討必須**：生の本音ログを外部APIへ送る機能なので、ユーザー同意・送信件数明示・オプトアウト導線をV2着手時に設計
- **音声アーカイブ（有料機能 / LIGHT以上）**
  - LIGHT以上のプランで、投稿時の生音声を保存する
  - 再生は**本人のみ**。タイムライン・他ユーザーには一切音声を出さない（テキストのみ表示）
  - 目的：テキストには残らない「その瞬間の声＝感情の記録」を残す。DAY 100にDAY 1の自分の声を聴ける体験
  - 技術：音声は Supabase Storage に保存。テキストは `public.records`（Supabase Postgres）
  - コスト設計：再生を本人限定にすることで egress（転送量課金）を最小化。課金者の分だけコストが発生する構造にする
  - 音声も編集・削除不可。刻印思想は音声にも適用される

## 12. 市場リサーチ（2026.05時点）

### 市場規模・トレンド

- デジタルジャーナルアプリ市場は2025年 約57億ドル → 2026年 約63億ドル（CAGR 11.4%）。2030年に約96億ドル見込み。

### 競合

- **国内：muute**（150万DL・Z世代中心）。ポジションは「癒し・整理・書く瞑想」。YUZUは意図的にこの逆（クール・挑発・記録）を張る。muuteが "肩を抱く" ならYUZUは "録音ボタンを突きつける"。
- **海外：Speakwise / AudioDiary / Rosebud / Day One** 等が音声ジャーナリングで乱立。ただし全社「整理・生産性・資産化」方向。YUZUだけが「整えるな・晒せ」で唯一の反整理ポジション。

### マクロインサイト（最大の追い風）

- Z世代の「反・デジタル完璧主義」運動（"2026 is the new 2016"）。BeReal は月間4,000万MAU、Z世代が約78%を占める。
- BeReal＝加工なしの「見た目」、YUZU＝加工なしの「声・思考」。同じ精神の音声版という空白ポジションを取れる。

### ポジショニング結論

- YUZUの空き地は「音声 × 反完璧主義 × 挑発的トーン」。muute（癒し系）も海外勢（生産性ツール）も来ない領域。競合が誰もいないのではなく、誰も来ようとしない領域に立っている。

## 13. 制約・設計思想

意図的に削ぎ落としている。

- 音声入力のみ（テキスト入力欄なし）
- 編集不可（テキスト・音声ともに）
- 削除不可（テキスト・音声ともに）
- フォロー / いいね / リポストなし
- **キャラクター方針（YUZU キャラ）**
  - **アプリ内**：キャラクターは登場させない。「THE RECORD」の世界観（信号・記録・番号）を守る。
  - **アプリ外（グッズ・SNS・Wear）**：IP資産としてキャラクターを保有・展開する。

  **キャラクター設定：**
  - シルエット：滑らかな縦長楕円（柚子の実）。凸凹・イボなし。
  - 色：`#F5D84A`（ゆず黄）ベース、`#1A1A2E`（墨）輪郭線
  - 表情：目のみ。口なし or 一直線。感情を読ませない。
  - スタンス：直立・飾りなし。余計なポーズをとらない。
  - タッチ：フラット2Dイラスト。BEANZライクなNFTアート的フラットベクター。グラデなし・影なし。
  - アクセサリー：基本なし。グッズ・Wear展開時のみINDEX番号を纏う。
  - キャラの性格：孤独・無口・芯がある。BEANZのストリート感に対し、YUZUキャラは「静かな確信」。

  **用途展開：**
  - LINEスタンプ（Phase 2）
  - YUZU Wear（投稿日数連動の購入権）
  - グッズ（ステッカー・ノート等）
  - SNS発信素材
- **ニックネーム / ユーザーアイコン / プロフィール表示なし**。SNS 機能を持たないため、identity は通し番号 `#NNN` のみ（v2）。
- **1日上限（フリープラン 3 回）はサーバ側で強制**（[app/api/records/route.ts](app/api/records/route.ts) の POST が `records` テーブルを JST 00:00 起点でカウントし、超過時 429 を返す）。複数端末でログインしてもカウントが同期される。localStorage は mock-mode 専用のフォールバック。
- **MARK（刻印）のみユーザーに許された能動操作**。編集・削除は依然不可。MARK は「お気に入り」ではなく、過去の声を本物として選び直す行為。
- **COPY（本文クリップボードコピー）は Notion 移行期間限定の一時機能**。将来削除する恒久 UI ではない（[components/RecordCard.tsx](components/RecordCard.tsx) に `// TEMPORARY:` コメント済）。
- **DM・音声メッセージ等の対人コミュニケーション機能を持たない**。YUZUのコアは「誰にも向けない声」であり、聞き手の存在が「整えるプレッシャー」を生む。思想と矛盾するため、意図的に除外する。

「整理する前」の生っぽい思考に価値があるという思想に基づく。

## 14. Eval（品質基準・テストケース）

> PRD の要件変更時は、対応する Eval ケースも同時に更新すること。

### STT（ElevenLabs Scribe v2）

**成功基準**：3秒以内・日本語として読める・NGタグ除去済み

- 30秒音声 → 句読点ありテキスト・3秒以内
- 無音・環境音 → 保存されない・フィードバック表示
- NGタグ → 除去されてから保存される
- 5文字未満 → 保存されない・フィードバック表示

### 感情分析（Claude API）

**成功基準**：自然な日本語・200文字以内・断定的ネガティブ表現なし

- ポジティブ投稿10件 → 前向きな要約
- ネガティブ投稿10件 → 断定的なネガティブ表現にならない
- NGケース：「あなたはネガティブです」→ 不合格

### モーダルUX

**成功基準**：録音→投稿→INDEX 反映が自然なフロー

- 待機中：浮遊ドットが軽快に動く
- 録音中：波形アニメーションが音量に反応する
- 完了後：`RECORDED.` ＋ 投稿テキスト
- INDEX に戻る：`#NNN` / RECORDS 一覧に反映されている

### コピートーン

**成功基準**：NGワード（癒し・寄り添う・育つ・香り・林・種・やさしく・ふんわり）がUI上に登場しないこと
