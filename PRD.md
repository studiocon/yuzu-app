# YUZU — PRD

> BE TRUE / 本物でいろ

YUZUは、声を絞り出すサービス。整えなくていい。うまく話さなくていい。長押し。話せ。あなたの声はそのまま記録になる。

---

## 1. ブランド

- **名前**：YUZU
- **タグライン**：BE TRUE
- **タグライン（日）**：本物でいろ
- **コアバリュー**：Raw. Real. You.
- **ミッション**：まだ知らない自分に、出会わせる。

### エレベーターピッチ

YUZU という名前は、果物の柚子から来ている。柚子は絞るか、削るか、熱を加えなければ香りが出ない。出さなければ、ただの黄色い球体だ。声も同じだ。外に出さなければ、自分は自分のまま留まる。

「BE TRUE」は、シェイクスピアの「To thine own self be true.」に由来する。YUZU が届けるのは命令ではなく、許可だ。あなたはすでに本物だ。ただ、出していないだけ。

YUZUは、声を絞り出すサービス。
整えなくていい。うまく話さなくていい。
長押し。話せ。
あなたの声はそのまま記録になる。

## 2. コンセプト・世界観

### 2.1 コアバリュー

```
Raw  → 加工しない・整理しない・ナマのまま
Real → 本物の自分・嘘のない声
You  → 主役はあなた自身
```

### 2.2 ポジショニング

```
muute → 心を癒す場所（やさしい・内省的・自己理解）
YUZU  → 本物を絞り出す装置（クール・挑発的・記録的）
SNS   → 他人に見せる場所
YUZU  → 自分を晒す場所
```

#### 3つの市場の隙間を狙う

1. **ジャーナリング市場（muute / Awarefy）**
   - 共通点：内省・自己理解
   - 違い：YUZUは「癒し」を持たない。挑発・命令・信号。muuteが "肩を抱く" ならYUZUは "胸ぐらを掴む"。

2. **SNS市場（X / BeReal）**
   - 共通点：思考や日常を外に出す
   - 違い：YUZUは他人に見せるためのものではない。晒しても評価されない構造。

3. **音声記録市場（Voice Memo / Otter / Untold）**
   - 共通点：音声で記録する
   - 違い：YUZUは "態度" を持つ。記録ツールではなく、自己と対峙する装置。

### 2.3 世界観「THE RECORD」

YUZU に庭はない。林もない。育つ比喩もない。
あるのは **記録（RECORD）** だけ。

```
声を出す → 信号（SIGNAL）になる
記録する → MARK（刻印）として残る
積み重なる → INDEX（番号体系）として並ぶ
```

声は加工されない信号。記録されることでしか自分は見えない。
装飾を削ぎ落とすこと自体が "Raw" の表現になる。

## 3. ターゲットユーザー

### 3.1 ターゲット階層

#### プライマリ：本音発信渇望層（Voice-Hungry）

- **定義**：20代〜30代。SNSに本音を出したいが、人目を気にして出せない層。
- **特徴**：
  - X鍵垢・Instagram裏垢を持っている
  - SNS疲れを感じている（Z世代の51%が該当 / SHIBUYA109調査）
  - 自分の発信が共感を得られるか不安がある（NRI調査）
  - muute・X・noteを使い分けているが、どれにも本音を出しきれていない
- **YUZUに来る動機**：誰にも見られず、誰にも評価されない場所で、整える前の声を残したい。

#### セカンダリ：自己理解探求層（Self-Discovery）

- **定義**：内省したいが、文章を書くハードルが高い層。
- **特徴**：
  - ジャーナリングを試したが3週間で挫折した経験がある
  - 「自分が何を考えているか分からない」自覚がある
  - 瞑想アプリ・モーニングページ・5年日記などを試している
  - muute・Awarefy・Day Oneを試したが続かなかった
- **YUZUに来る動機**：書く構えを取らずに、声で思考を外に出したい。

#### ターシャリ：クールカルチャー層（Street / Culture）

- **定義**：カルチャー感度の高い20代〜30代。NIKE・Patta・A-COLD-WALL\*・Supreme・Off-Whiteなどに親和性がある層。
- **特徴**：
  - 「やってる感」を嫌う一方で、本物の習慣には価値を置く
  - メンタル系・ウェルネス系アプリは「ダサい」と感じる
  - StravaやNIKE Run Clubは続けている
  - YUZU INDEX・YUZU Wearに反応する
- **YUZUに来る動機**：「これ使ってる」と言えるブランドだから。INDEX番号やSTREAKで自分のストイックさを可視化したい。

### 3.2 除外ターゲット

YUZUが意図的に取りに行かない層。

- **30代後半以上のウェルネス層** → muuteで満たされている
- **本格的メンタル不調層** → 専門治療（Awarefy・カウンセリング）が必要
- **きれいな日記を書きたい層** → 編集削除不可と相性最悪
- **承認欲求が主動機の層** → いいね・フォローがないと続かない

### 3.3 ペルソナ

#### ペルソナA：ナナミ（27歳・編集者）「言葉を持て余す内省者」

- **属性**：都内・出版社勤務・独身・千駄ヶ谷在住
- **行動特性**：X鍵垢、Instagram裏垢、TikTok閲覧専。ずとまよ、ベイクルーズ、ATELIER NAVE
- **悩み**：思考を文章にする職業なのに、自分の本音は書けない。"言葉になる前の感情" を逃している感覚がある。
- **インサイト**：「きれいに書こうとした瞬間、嘘になる」のを知っている。だから書けない。
- **利用シーン**：仕事帰り、駅から家までの15分。整える前の自分を、捨てずに残しておきたい。

#### ペルソナB：リク（24歳・グラフィックデザイナー）「クールに自分を晒したい挑発者」

- **属性**：フリーランス・代々木上原・MacBook Air
- **行動特性**：Patta, NIKE SB, A-COLD-WALL\*, Travis Scott。『ブルータス』『POPEYE』『IDEA』『装苑』を読む
- **悩み**：ジャーナリングは陰キャっぽい。でも自分と向き合う時間は欲しい。「やってる感がダサい」「本物かどうか」が判断軸。
- **インサイト**：「これ使ってる」と言えるブランドなら、する。
- **利用シーン**：深夜の制作中、作業の合間。アイデアとも愚痴ともつかない声を吐き出して、INDEX番号を増やすのが快感。

#### ペルソナC：ユイ（31歳・人事）「ジャーナリングに挫折した自己探求者」

- **属性**：中堅メーカー人事・横浜・結婚2年目
- **行動特性**：muute、5年日記、モーニングページ、瞑想アプリ — すべて3週間で挫折
- **悩み**：「ちゃんと書こう」としてしまう。
- **インサイト**：続かないのは意志が弱いからではない。「ちゃんとやろう」という構えが続かない原因だと薄々気づいている。
- **利用シーン**：朝のメイク中、洗濯機を回している間、寝る前のベッド。両手が塞がっている時間に、声だけで残せることが革命的。

### 3.4 インサイト

#### インサイト1：整理した瞬間、本物じゃなくなる

- **主張**：文章は推敲する。推敲とは無意識のうちに "見せたい自分" に近づける作業。
- **根拠 / 背景**：声は推敲できない。口から出た瞬間、嘘がつけない。これがYUZUの最大の発明。

#### インサイト2：キーボードは構えの装置

- **主張**：スマホを開く → アプリ起動 → キーボードが立ち上がる、この時点で人は "書く構え" に入っている。
- **根拠 / 背景**：長押し録音は、構えを与えない。思考が言語化される前の、皮膚感覚の声が拾える。

#### インサイト3：日記が続かないのは、形式があるから

- **主張**：形式（プロンプト、テンプレ、AIの励まし）があると、人は "正解" を探し始める。
- **根拠 / 背景**：YUZUは形式を持たない。プロンプトすら「整えるな。話せ。」と命令形で剥がす。既存のジャーナリングアプリ（muute等）は、ポジティブ報告を促すフィードバックでユーザーを追い詰めるケースがある。YUZUはフィードバックで誘導しない。

#### インサイト4：やってる自分がイケてる、と思いたい

- **主張**：NIKE TRAINING CLUB、Strava、NIKE Run Clubが流行ったのは「やっている」状態が可視化できたから。
- **根拠 / 背景**：YUZUのSTREAK・DAY・RECORDSはストイックさの可視化装置。muuteの "やわらかい数字" とは正反対の方向に振る。

#### インサイト5：声は最も親密で、最も嘘がつけない媒体

- **主張**：電話で泣くのは止められない。声には体温がある。
- **根拠 / 背景**：自分の声を再生して聴くのは、文字を読み返すのとは別次元の自己対峙。これがV2以降のAIインサイト読み上げ（TTS）の本質的価値。

### 3.5 ペイン

| ペイン | 既存サービスの状態 | YUZUの解 |
|---|---|---|
| 書く前にやめてしまう | キーボード起動の摩擦 | 長押しだけ |
| 上手く書こうとしてしまう | 編集機能、保存ボタン | 編集削除不可 |
| 続かない | カレンダー型UI、リマインダー | STREAK・INDEX・モノ展開 |
| SNSで本音が出せない | 人目、フォロワー、いいね | 晒しても評価されない構造 |
| 自分の感情が分からない | テキスト分析のみ | 声の波形・トーン（V2以降の余地） |

特に "上手く書こうとしてしまう" は他では解けない。テキスト入力できる時点で、人は構えてしまう。YUZUがテキスト入力欄を持たないのは、思想として正しい。

### 3.6 Jobs To Be Done

#### Functional Job（機能的）：整理する前の思考を、捨てずに残したい

- 移動中・歩行中・両手が塞がっている時に記録したい
- 言語化する手間なく、感情を外に出したい
- 後で振り返れる形でストックしたい

#### Emotional Job（感情的）：整える前の自分を、許可したい

- 本音を出していい場所が欲しい
- "ちゃんとしてない自分" を否定されたくない
- SNSの "見せ場" から退避したい
- 自分と対話している感覚が欲しい

#### Social Job（社会的）：自分を律してる人間でありたい / そう見られたい

- 続けてる自分がかっこいいと思いたい
- "メンタル系アプリ" ではなく "ストイックなツール" として持ちたい
- INDEX番号やYUZU Wearで、努力の可視化・自己表現したい

この三層のうち、最も差別化できるのはSocial Job。muuteは "癒し" 止まりでSocialまで行けていない。YUZUのINDEX→Goods→Wear展開は、Social Jobを満たす設計として機能する。NIKEがランニングを "かっこいいこと" にしたのと同じ構造。

| 比喩 | 自然・植物・育つ | 信号・記録・刻印 |

### 4.2 ビジュアル

```
muute   : パステル・水彩・余白・キャラクター
Awarefy : ブルー・医療的・グラフ・専門性
YUZU    : 単色オフホワイト + 信号黄 + 黒・グリッド・タイポ駆動
```

参照軸：Swiss International Typographic Style、NIKE、Patta、A-COLD-WALL\*、Off-White、Acne Studios。
これはジャーナリングアプリのリファレンスではなく、ストリート / ハイファッション / モダニズムの参照群。

### 4.3 プロダクトの態度

- **muute** = サポーター（隣で励ます）
- **YUZU** = ミラー（突きつける）

ミラーは答えを返さない。自分の声を、自分が聴く。これだけ。介在しない潔さがブランド。

### 4.4 拡張性

```
muute   → アプリで完結
Awarefy → 法人 / 教育機関へ
YUZU    → IP化・グッズ・アパレル・書籍・音楽
```

YUZU INDEX、YUZU Goods、YUZU Wear、YUZU Story、YUZU Sound という拡張軸を持つ。「投稿日数で番号が解放される」設計は、Supremeのドロップ文化 × Stravaの記録性 × NIKE Runの達成感を統合する構造。アプリ単体ではなく、カルチャーとして展開する。

YUZUは "自己肯定" を一切しない。「あなたは本物だ」と励まさない。ただ「許可する」だけ。評価しないことが、最大の許可。

世のセルフケア系サービスの大半は "あなたを肯定する" モデル。YUZUは "あなたを既に本物として扱う" モデル。前提が違う。

シェイクスピア「To thine own self be true.」（自分自身に本物であれ）の現代的解釈として、YUZUは命令ではなく許可を届ける。

## 6. 機能要件（MVP v1）

### 基本構成

- **3タブ構成**：タイムライン / はなす（デフォルト）/ マイページ。下部に iOS 26 Liquid Glass 風のフローティングタブバー。
- **ゲストモード**：投稿・閲覧・リアクション可能（cookie で sessionId を保持）。

### 投稿

- マイクボタン長押し中のみ録音（MediaRecorder）
- 離した瞬間に自動で STT（ElevenLabs Scribe v2）送信
- テキスト変換後、自動投稿（生音声は保存しない）
- 編集・削除不可
- テキスト入力欄なし
- NGタグ（[background noise] 等）を除去してから保存
- 5文字未満の発話は破棄しフィードバック表示

### マイページ（デフォルト画面）

- ニックネーム登録・編集（未登録はランダム果物名）
- 果物絵文字アイコン（既存資産・MVP では維持）
- スタッツ表示（英語ラベル）：
  - **DAY ○** — 登録からの日数
  - **○ RECORDS** — 投稿数
  - **○ DAYS. NO SKIP.** — 連続投稿日数
  - **LATEST #○** — 最新のINDEX番号
  - **○ CHARS** — 総文字数
  - **○ MIN** — はなした時間（投稿数 × 平均30秒）
- 感情分析チャート（Claude API・折れ線グラフ）
- 投稿一覧（新着順）

### INDEX番号

ユーザーごとの投稿通し番号。話すたびに1ずつ増加する個人の記録番号。

**ルール**
- 1始まり・ゼロ埋めなし（`#1` `#247` `#10000`）
- 削除・編集不可のため欠番は発生しない
- 上限：999999（6桁）
- 呼称「インデックス番号」はUI上に出さない。`#` + 数字のみ。

**表示箇所**

| 場所 | 形式 | スタイル |
|---|---|---|
| 投稿カード（タイムライン） | `#N · X時間前` | Unbounded 700 / `--ink-muted` / 11px |
| 投稿完了モーダル | `RECORDED.` の下に `#N` | Unbounded 700 / `--ink` / 32px |
| マイページ スタッツ | `LATEST` ラベル + `#N` | 既存 `.mypage-stat-card` に準拠 |
| 週次レポート | `WEEK #N` | Unbounded 700 / 18px（登録日からの週数） |
| 月次レポート | `MONTH #N` | Unbounded 700 / 18px（登録日からの月数） |

**Redisキー設計**

```
user:{sessionId}:index      // 現在のINDEX番号（INCR で採番）
```

WEEK・MONTHは登録日（最初の投稿日）からの経過で自動算出（保存不要）。

**Eval**
- 投稿後、完了モーダルに `#N` が表示される
- タイムラインの各投稿に `#N · X時間前` が表示される
- マイページ LATEST スタッツが更新される
- `INDEX` `No.` 等の文字列がUI上に登場しない
- ゼロ埋め（`#001`）が発生しない

### 投稿促進プロンプト

マイクボタン上にランダム表示（毎セッションで変わる）：

```
長押し。話せ
何が本当なのか？
言ってないことは？
今日、何があった？
本当はどう思ってる？
何から逃げてる？
何に怒ってる？
誰にも言えないことを
整えるな。話せ
1分でいい。出して
```

### マイクボタン

- 右下フローティング（FAB）
- 全画面から常時表示
- タップでモーダル起動

### モーダル（録音画面）

- 背景：**オフホワイト単色** `#FAFAF5`
- 上半分：アニメーションエリア
  - 待機中：**浮遊するドット**（既存 `float-dot` 維持）
  - 録音中：**音量に反応する波形**（既存波形アニメ維持）＋ 同心円リング
  - 変換中：ドットが中心に集まり消える
- 下部：マイクボタン（正円。歪んだ楕円シェイプは廃止）
- 録音完了後：**`RECORDED.`** ＋ 投稿テキスト → ホームに戻る

### UIコピー

英語＝状態ラベル（Unbounded）／日本語＝プロンプト・本文・エラー（LINE Seed JP）として使い分ける。

| 場所 | コピー |
|---|---|
| 待機（マイクボタン下） | 長押し。話せ |
| 録音中 | RECORDING. |
| 変換中 | DECODING. |
| 投稿完了 | RECORDED. |
| タイムライン空 | 最初の一声を / はじまりを、語って / ここから始まる（ランダム） |
| ニックネーム未設定 | 名前をつけよう |
| 投稿数 | ○ RECORDS |
| 連続日数 | ○ DAYS. NO SKIP. |
| 登録日数 | DAY ○ |
| 音声なし | 声が聞こえなかった。もう一度。 |
| 短すぎ | 短い。もう一度。 |
| 投稿完了・INDEX表示 | #○ |
| スタッツ・最新INDEX | LATEST |
| 週次レポートラベル | WEEK #○ |
| 月次レポートラベル | MONTH #○ |

### SIGNALカード

番号＋日付＋STREAKの「刻印」として設計するシェアアイテム。
投稿内容を晒さずに「自分がここにいる」を証明できる。
Xなどへのシェアがネットワーク効果の起点になる。

**構成要素**
- `YUZU #○○` — YUZU INDEX番号（アイデンティティの核）
- `YYYY.MM.DD` — 刻印日（「この日に存在した」事実）
- `○○ DAYS. NO SKIP.` — STREAK（継続の証拠）

**解放条件**
- LIGHT以上：基本SIGNALカード
- PREMIUM：全バリエーション解放
- STREAK達成時：限定デザイン

## 13. Eval（品質基準・テストケース）

> PRD の要件変更時は、対応する Eval ケースも同時に更新すること。

### STT（ElevenLabs Scribe v2）

**成功基準**：3秒以内・日本語として読める・NGタグ除去済み

- 30秒音声 → 句読点ありテキスト・3秒以内
- 無音・環境音 → 保存されない・フィードバック表示
- NGタグ → 除去されてから保存される
- 5文字未満 → 保存されない・フィードバック表示

### 感情分析（Claude API）

**成功基準**：自然な日本語・200文字以内・断定的ネガティブ表現なし

- ポジティブ投稿10件 → 前向きな要約
- ネガティブ投稿10件 → 断定的なネガティブ表現にならない
- NGケース：「あなたはネガティブです」→ 不合格

### モーダルUX

**成功基準**：録音→投稿→ホーム復帰が自然なフロー

- 待機中：浮遊ドットが軽快に動く
- 録音中：波形アニメーションが音量に反応する
- 完了後：`RECORDED.` ＋ 投稿テキスト
- ホームに戻る：マイページ／タイムラインに反映されている

### コピートーン

**成功基準**：NGワード（癒し・寄り添う・育つ・香り・林・種・やさしく・ふんわり）がUI上に登場しないこと
